# Сравнение аналогов
## Принцип отбора аналогов

Поиск аналогов open-source инструментов ГИС и обработки топографических данных проводился с помощью GitHub Search, Google Search, Google Scholar и специализированных каталогов (OSGeo.org, Awesome-Geospatial) с запросами:

- open source GIS tools
- open source geospatial analysis software
- open source DEM processing tools
- GRASS GIS alternatives
- QGIS topography plugins
- open source LiDAR processing
- OSGeo projects

Дополнительно анализировались разделы "Tools & Resources" [1], каталог github [2] и списки репозиториев в найденных проектах вручную.

Были отобраны проекты с активной разработкой (обновления 2024–2025 гг., stars >1k), лицензией GPL/MIT, кросс-платформенностью и поддержкой растровых данных/DEM, исключая коммерческие и устаревшие решения. Описанные инструменты применимы к анализу топографии и ГИС-данных в целом. Они охватывают разные аспекты обработки: визуализацию, анализ рельефа.
### GRASS

GRASS - это вычислительный движок для растровой, векторной и геопространственной обработки. Он поддерживает моделирование местности и экосистем, гидрологию, управление данными и обработку изображений. Благодаря встроенному временному фреймворку и Python API он обеспечивает расширенный анализ временных рядов и геопространственное программирование, подходящее для крупномасштабного анализа на различных аппаратных конфигурациях [3].

### WhiteboxTools

WhiteboxTools - это платформа для анализа геопространственных данных.
WhiteboxTools можно использовать для выполнения общих операций по анализу географических информационных систем (ГИС), таких как анализ расстояния затрат, буферизация раста и растровая реклассификация. WhiteboxTools также содержит расширенные инструменты для пространственного гидрологического анализа (например, накопление потока, размеление водоразделов, анализ сети потоков, удаление раковины), анализа местности (например, общие индексы местности, такие как наклон, кривизны, индекс влаги, затенение холмов; гипсометрический анализ; многомасштабный топографический анализ положения) и обработки данных LiDAR [4].

### OpenTopography

OpenTopography — это платформа, которая обеспечивает упрощенный и открытый доступ к высококачественным топографическим данным от лидара. Сюда входят наборы данных глобального масштаба средней точности, а также высококачественный лидар от источников вроде 3D Elevation Program Геологической службы США, прибрежного лидара NOAA и Land Information New Zealand. Эти данные можно применять для снижения природных рисков, таких как оползни и землетрясения, для понимания рек и потенциальных рисков наводнений, проектирования коммунальной и транспортной инфраструктуры, а также для документирования созданной человеком среды и культурного наследия [1].

### QGIS

QGIS — это бесплатная открытая ГИС для просмотра, редактирования, анализа геоданных. Поддерживает векторные, растровые слои, облака точек, интегрируется с SAGA и плагинами на Python, может работать с форматами по типу shapefile, PostGIS, WMS. Работает на Windows, Linux, macOS, используется учеными, инженерами для анализа 3D-моделей рельефа. Есть мобильная версия QField для работы в полевых условиях [5].

### xDEM

xDEM — это открытый проект по созданию основного Python-пакета для анализа цифровых моделей рельефа (DEM). Он нацелен на предоставление модульных и устойчивых к нагрузкам инструментов для самых распространенных анализов с DEM, включая геопространственные операции специфичные для DEM и широкий спектр методов 3D-выравнивания и коррекции из опубликованных рецензируемых исследований.  Также xDEM включает продвинутые инструменты анализа неопределенностей на основе пространственной статистики из SciKit-GStat [6].

## Критерии сравнения аналогов

### Анализ топографических индексов

Анализ топографических индексов — показатель алгоритмов предназначенных для расчета основных показателей рельефа из DEM: наклон, аспект и кривизна профиля/плана.
Этот критерий важен для анализа рельефа местности, так как индексы учитываются в расчетах в гидрологии, экологии и геоморфологии. Измерение осуществляется путем сбора данных о том, какое количество индексов могут проанализировать и предоставлять ПО.

### Кросс-платформенность

Кросс-платформенность — стабильная работа на основных ОС (Windows, Linux, macOS) и мобильных устройствах без обязательного использования виртуализации (Docker/WSL).
Критерий важен для команд с разными ОС, так как при сложном портировании ПО предпочтение ему отдаваться не будет. Измерение осуществляется путем сбора данных о колличестве ОС, в которых это решение будет работать (Windows, Linux, macOS).

### Размер сообщества
Размер сообщества - это параметр, который будет отражать то, насколько люди вовлечены в продукт и насколько велика активная аудитория.
Критерий важен потому, что он отражает доступность поддержки со стороны сообщества в решении проблем. Измерение осуществляется путем сбора параметров Stack Overflow questions, GitHub Stars, GitHub Forks.

## Выводы по итогам сравнения

WhiteboxTools показал самое большое значение по количеству топографических индексов — 59 штук. Этот параметр является необходимым для узкопрофильных исследований в геоморфологии. В то же время показатели QGIS и GRASS (15 и 22 соответственно) дают достаточный уровень для решения ординарных задач в анализе рельефа.
QGIS, GRASS, WhiteboxTools и xDEM совместимы с основными ОС (Windows, Linux, macOS), в то время как OpenTopography является веб-приложением, что делает его доступным практически на всех этих системах. Из этого можно сделать вывод, что все решения практически одинаковы по уровню гибкости развертывания их на разных ОС.
По результатам показателя GitHub Stars лидирующую позицию занимает QGIS, опережая остальных в 10 раз и более, что показывает то, на сколько выше количество пользователей выбирают именно это решение.
Схожие цифры можно увидеть и в сравнении показателя GitHub Forks. QGIS имеет 3,3 тысячи, что больше чем у всех конкурентов в сумме. Этот показатель отражает то, что сообщество QGIS дает новым пользователям больше модифицированных версий для более точечного использования.
Показатель questions Stack Overflow коррелирует с показателями GitHub Forks и GitHub Stars. У QGIS этот показатель 1670 против 80 у конкурентов, что отражает большой размер сообщества, но иногда, для узкопрофильных продуктов этот показатель может быть не велик, так как все важные аспекты могут быть отражены в документации.
По итогу сравнения и анализа его результатов, можно сделать вывод, что нет прямой зависимости между аналитическими возможностями и экосистемными. То есть большое количество индексов необходимо для специальных задач, но развитое сообщество облегчает доступ к готовым решениям и примерам, что позволит сформулировать требования к новому решению с ориентацией на сбалансированные параметры.

## Выбор метода решения

На основе анализа аналогов в таблице сравнения выявлены ключевые требования к разрабатываемому ПО: среднее количество поддерживаемых индексов (как у QGIS и xDEM — 12-15), кросс-платформенность на трех ОС (Windows, Linux, macOS, как у QGIS, GRASS, WhiteboxTools и xDEM) и популярность для удобства освоения (учитывая GitHub-метрики и вопросы на Stack Overflow).
Решение должно быть ориентировано на создание программного комплекса, который:

* Обеспечивает высокую точность геоморфометрических вычислений (на уровне WhiteboxTools) с расчетом уклона по методу Хорна (3x3 окно), аспекта, кривизны, фильтрацией шумов (median blur), pit filling и многопоточной обработкой (joblib) для DEM > 1 км² (RMSE < 0,5° на тестовых полигонах).
* Поддерживает кросс-платформенность без установки ГИС.
* Обеспечивает простоту доступа к данным как у OpenTopography и интеграцию с Telegram-ботом .

Такой подход обеспечивает превосходство над аналогами по комбинации точности, переносимости и удобства для неспециалистов, с модульной архитектурой для расширения.

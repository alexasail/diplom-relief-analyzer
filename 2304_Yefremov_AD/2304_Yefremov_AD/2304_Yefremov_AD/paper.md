# Разработка бота в телеграм для автоматизированного анализа цифровых моделей рельефа.

Ключевые слова: рельеф местности, цифровая модель рельефа (DEM), кроссплатформенность.

## Аннотация

В статье рассмотрена разработка телеграм-бота для автоматизированного анализа цифровых моделей рельефа (DEM) неспециалистами без установки ГИС-систем. Проведен обзор пяти open-source инструментов: GRASS, WhiteboxTools, OpenTopography, QGIS, xDEM — где WhiteboxTools выделяется максимальным набором топографических индексов, а QGIS лидирует по активности сообщества. Разработана архитектура ReliefAnalyzer, которая загружает высокоточные DEM-данные из OpenTopography, применяет медианную фильтрацию для устранения шумов, вычисляет уклоны методом Хорна и формирует наглядную визуализацию в Matplotlib с картами высот и уклонов. Бот на базе aiogram обеспечивает простой интерфейс: пользователь отправляет координаты и мгновенно получает готовое PNG-изображение с легендами и подписями. Docker-контейнер с локальным кэшем гарантирует кросс-платформенный запуск без дополнительных настроек. Решение снижает порог входа для анализа рельефа, сочетая точность профессиональных инструментов с доступностью мессенджера.


## Введение

По данным на 2024 год, некорректный анализ рельефа приводит к дополнительным затратам на проектирование объектов недвижимости[1]. На текущий день нет общепринятых методов автоматизированного анализа рельефа местности для неспециалистов. Рассмотренные решения для анализа рельефа местности, являющиеся одними из самых распространенных, ограничивались либо загруженностью интерфейса, либо низким качеством результатов[2]. Например, профессиональные ГИС-системы (ArcGIS) требуют годового обучения и оплаты, которая недоступна в РФ, что делает их крайне трудозатратными для использования[3]. В то время как онлайн-решения (Google Earth, OpenTopoMap) не дают количественного анализа уклонов и высот, такие сервисы представляют только визуализацию[4].
Измерение параметров рельефа затруднено различными ограничениями. Аналитические работы в области рельефа местности являются развитой областью науки. Знания получены в трудах Кошкарева А.В.[5] и Набиева А.А.[6], что дает теоретическую основу для разработки новых алгоритмов и методов. Предлагаемое решение к анализу рельефа может снизить время анализа, а также понизит порог входа, за счет использования мессенджера, которым пользуются 73% россиян, что повысит скорость работы и затраты на квалифицированные кадры[7]. Разработка аналитических решений для пространственных данных, в том числе геодезических, имеет поддержку государства, через программы, например "Цифровая экономика Российской Федерации"[8], что в очередной раз подчеркивает актуальность разработки. Помимо этого, данная область программного обеспечения будет иметь годовой рост 13,7% вплоть до 2030 года[9].
Неспециалист в области анализа рельефа местности — это человек, не имеющий профессионального образования или специальной подготовки в области анализа рельефа, но нуждающийся в своей профессиональной деятельности в результатах такого анализа.
Для решения задачи анализа рельефа местности была поставлена цель разработать бот в мессенджере для анализа рельефа и высот местности.
Для достижения поставленной цели необходимо:
- Провести исследования существующих решений
- Разработать последовательность операций по построению карт высот и уклонов рельефа
- Разработать архитектуру телеграм-бота для получения результатов работы алгоритма


Объектом исследования являются алгоритмы анализа рельефа местности. Предметом исследования является скорость обработки данных и вычисления результатов.

## Обзор предметной области

### Принцип отбора аналогов

Поиск аналогов open-source инструментов ГИС и обработки топографических данных проводился с помощью GitHub Search [10], Google Search, Google Scholar и специализированных каталогов (OSGeo.org [11]) с запросами:

- open source GIS tools
- open source geospatial analysis software
- open source DEM processing tools
- GRASS GIS alternatives
- QGIS topography plugins
- open source LiDAR processing
- OSGeo projects

Дополнительно анализировались разделы "Tools & Resources" [12], каталог github [13] и списки репозиториев в найденных проектах вручную.

Были отобраны проекты с активной разработкой (обновления 2024–2025 гг., stars >1k), лицензией GPL/MIT, кроссплатформенностью и поддержкой растровых данных/DEM, исключая коммерческие и устаревшие решения. Описанные инструменты применимы к анализу топографии и ГИС-данных в целом. Они охватывают разные аспекты обработки: визуализацию, анализ рельефа.
#### GRASS

GRASS - это вычислительный движок для растровой, векторной и геопространственной обработки. Он поддерживает моделирование местности и экосистем, гидрологию, управление данными и обработку изображений. Благодаря встроенному временному фреймворку и Python API он обеспечивает расширенный анализ временных рядов и геопространственное программирование, подходящее для крупномасштабного анализа на различных аппаратных конфигурациях [14].

#### WhiteboxTools

WhiteboxTools - это платформа для анализа геопространственных данных.
WhiteboxTools можно использовать для выполнения общих операций по анализу географических информационных систем (ГИС), таких как анализ расстояния затрат, буферизация растра и растровая реклассификация. WhiteboxTools также содержит расширенные инструменты для пространственного гидрологического анализа (например, накопление потока, размеление водоразделов, анализ сети потоков, удаление раковины), анализа местности (например, общие индексы местности, такие как наклон, кривизны, индекс влаги, затенение холмов; гипсометрический анализ; многомасштабный топографический анализ положения) и обработки данных LiDAR [15].

#### OpenTopography

OpenTopography — это платформа, которая обеспечивает упрощенный и открытый доступ к высококачественным топографическим данным от лидара. Сюда входят наборы данных глобального масштаба средней точности, а также высококачественный лидар от источников вроде 3D Elevation Program Геологической службы США [16], прибрежного лидара NOAA[17] и Land Information New Zealand[18]. Эти данные можно применять для снижения природных рисков, таких как оползни и землетрясения, для понимания рек и потенциальных рисков наводнений, проектирования коммунальной и транспортной инфраструктуры [12].

#### QGIS

QGIS — это бесплатная открытая ГИС для просмотра, редактирования, анализа геоданных. Поддерживает векторные, растровые слои, облака точек, интегрируется с SAGA и плагинами на Python, может работать с форматами по типу shapefile[19], PostGIS[20], WMS[21]. Работает на Windows, Linux, macOS, используется учеными, инженерами для анализа 3D-моделей рельефа. Есть мобильная версия QField для работы в полевых условиях [22].

#### xDEM

xDEM — это открытый проект по созданию основного Python-пакета для анализа цифровых моделей рельефа (DEM). Он нацелен на предоставление модульных и устойчивых к нагрузкам инструментов для самых распространенных анализов с DEM, включая геопространственные операции специфичные для DEM и широкий спектр методов 3D-выравнивания и коррекции из опубликованных рецензируемых исследований.  Также xDEM включает продвинутые инструменты анализа неопределенностей на основе пространственной статистики из SciKit-GStat [23].

### Критерии сравнения аналогов

#### Анализ топографических индексов

Анализ топографических индексов — показатель алгоритмов предназначенных для расчета основных показателей рельефа из DEM: наклон, аспект и кривизна профиля/плана.
Этот критерий важен для анализа рельефа местности, так как индексы учитываются в расчетах в гидрологии, экологии и геоморфологии. Измерение осуществляется путем сбора данных о том, какое количество индексов могут проанализировать и предоставлять ПО.

#### Кроссплатформенность

Кроссплатформенность — стабильная работа на основных ОС (Windows, Linux, macOS) и мобильных устройствах без обязательного использования виртуализации (Docker/WSL).
Критерий важен для команд с разными ОС, так как при сложном портировании ПО предпочтение ему отдаваться не будет. Измерение осуществляется путем сбора данных о количестве ОС, в которых это решение будет работать (Windows, Linux, macOS).

#### Размер сообщества
Размер сообщества - это параметр, который будет отражать то, насколько люди вовлечены в продукт и насколько велика активная аудитория.
Критерий важен потому, что он отражает доступность поддержки со стороны сообщества в решении проблем. Измерение осуществляется путем сбора параметров Stack Overflow questions, GitHub Stars, GitHub Forks.

## Выбор метода решения

На основе анализа аналогов выявлены ключевые требования к разрабатываемому ПО: среднее количество поддерживаемых индексов (как у QGIS и xDEM — 12-15), кроссплатформенность на трех ОС (Windows, Linux, macOS, как у QGIS, GRASS, WhiteboxTools и xDEM) и популярность для удобства освоения (учитывая GitHub-метрики и вопросы на Stack Overflow).
Решение должно быть ориентировано на создание программного комплекса, который:

* Обеспечивает высокую точность геоморфометрических вычислений (на уровне WhiteboxTools) с расчетом уклона по методу Horn (3x3 окно), аспекта, кривизны, фильтрацией шумов (median blur).
* Поддерживает кроссплатформенность без установки ГИС.
* Обеспечивает простоту доступа к данным как у OpenTopography и интеграцию с телеграм-ботом.

Такой подход обеспечивает превосходство над аналогами по комбинации точности, переносимости и удобства для неспециалистов, с модульной архитектурой для расширения.

## Описание метода решения

Архитектура телеграм-бота заключается в последовательном выполнении операций в основном классе ReliefAnalyzer, принимающем в себя координаты широты и долготы. Далее он генерирует карту рельефа с соответствующими метриками. И возвращает полученные результаты пользователю. Схема работы бота изображена на рис. 1.
![Рисунок 1 - Архитектура телеграм-бота](./architecture.png)

Рисунок 1 - Архитектура телеграм-бота

Коммуникация пользователя и программы спроектирована через телеграм-бота на основе библиотеки aiogram [30]. Бот принимает сообщение от пользователя с координатами, обрабатывает его, вычисляет координатную рамку и передает данные в анализатор. Далее получает от анализатора ответ в виде изображения в формате PNG.
ReliefAnalyzer обрабатывает данные в несколько этапов. На первом этапе происходит загрузка цифровой модели рельефа из OpenTopography с последующим анализом и формированием NumPy-массива [31]. Второй шаг состоит из применения медианного фильтра, для того чтобы избавиться от шумов. Далее происходит процесс расчёта уклонов по методу Хорна(используются градиенты в окне 3х3).
Matplotlib создает визуализацию путем формирования двух полей. Первое с высотами в цветовой схеме "terrain". Второе с уклонами в цветовой схеме "coolwarm". Далее визуализация дополняется цветовой шкалой и подписями координат. Финальная визуализация сохраняется в PNG.
Код структурирован по четырем модулям: analyzer.py, bot.py, parse.py и main.py. Отсутствие зависимостей от ГИС-систем гарантирует высокую переносимость.
Пользователю в боте будет доступна команда "Построить карту высот и уклонов". После чего, пользователю будет предложено в качестве текстового сообщения отправить координаты местности, карту высот и уклонов которой он хочет получить. После успешной работы пользователь получает PNG файл в котором слева расположена карта высот, а справа карта уклонов. Все карты сопровождены соответствующими легендами, для расшифровки и полного понимания.
## Заключение

В ходе исследования достигнута цель: разработана архитектура телеграм-бота для автоматизированного анализа цифровых моделей рельефа (DEM) неспециалистами. 

Проведен обзор пяти open-source ГИС-инструментов и их сравнение по топографическим индексам, кросс-платформенности и размеру сообщества. Из этого можно сделать вывод, что нет прямой зависимости между аналитическими возможностями и экосистемой — QGIS лидирует по сообществу, а WhiteboxTools — по индексам. Итогом проведенного анализа является создание сбалансированных требований к новому решению. Это позволяет избежать переусложнения и ориентироваться на удобство для неспециалистов, снижая риски внедрения.

Разработана последовательность операций включающая в себя: методом Хорна, медианной фильтрации шумов и визуализации в Matplotlib. Это позволяет анализировать рельеф без ошибок от шумов, минимизируя затраты на проектирование.

Спроектирована модульная система на aiogram с загрузкой DEM из OpenTopography, интеграцией NumPy и Docker-контейнером для кросс-платформенности с локальным кэшем. Что обеспечивает переносимость без установки ГИС. Это понижает порог входа для неспециалистов. Недостатком разработанного решения является зависимость от внешних API.
